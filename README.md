# 总体思路阐述
数据预处理部分内容在下文有详细的阐述这里不再重复。

项目是分类问题，而且是类别不平衡的、代价敏感的问题 对于这个问题我们从两个方向上来解决
  1. 特征工程
     - 这两步后续会做消融实验来验证两者的贡献
     - 1. 基于 Boruta 的特征选择（去伪/剔除噪声干扰）：同分布下随机打乱特征顺序做成影子变量 使用随机森林进行分类 观察原始特征是否显著优于影子变量 用以特征选择
     - 2. 基于 SDAE 的深度特征表示（存真/增强）：鉴于原始特征可能存在稀疏或非线性关系弱的问题 参考了SDAE形成特征表示 原理：对于数据集进行随机mask 训练模型进行完形填空 期待模型能够通过自身不断提取的高级特征来理解数据内部稳健的潜在特征表示 然后我们将模型每层拆开 提取网络各隐藏层输出的垂直特征 作为高阶抽象特征输入到分类器中 以丰富特征空间的信息量
  2. 数据均衡
      - 同样也是两个手段 尝试混合的效果
      - 1. 数据层面的增强 SMOTE:沿着近邻样本连线方向进行的线性插值的方法 用以扩展少数类的数据量
      - 2. 算法层面的融合 bagging：传统文献都是采用过采样少数类 欠采样多数类做到数据均衡的 但是这种方法可能会丢失信息或引入噪声
      我希望通过SMOTE（线性插值 本身不改变分布）+bagging的方式让模型尽可能多的学习到少数类信息 与此同时在每个bagging子集上 使用改写的代价函数（代价敏感）深度神经网络训练 通过集成学习的投票机制 不仅能保留更多多数类样本的信息 还能通过调整误分类惩罚权重 显著提升模型对少数类（高价值样本）的敏感度

在可解释性上，首先使用shap进行局部分析（就我的建模经验来讲shap的解释一般是不太靠谱的）接下来使用SEA/HotFlip（目前来看模型是灰盒模型 最终选择哪个还是看哪个好实现）通过反事实的测试 选取 SHAP 判定为‘高重要性’的特征 对其进行微小的对抗性扰动或反事实修改 观察修改这些被 SHAP 标记为关键的特征，是否能以最小的代价促成预测结果的翻转（从侧面证明shap进行事后解释的合理性） 总的来说就是用 SHAP 找重点 用反事实（HotFlip）来测谎

  - **缺失机制检验**：
    - 应用Little's MCAR检验判断缺失机制（MCAR/MAR/MNAR）
      - (如果我想用**贝叶斯岭/ML算法**进行缺失值补全 首先我要检验数据缺失情况是不是随机的 缺失情况不能与特征和标签有关：例如若总费用越高缺失越严重就需要考虑原因了 不能轻易进行缺失值补全)
    - 检验缺失指示变量与其他观测变量的关系

  - **缺失值处理策略**：
    - 若检验结果支持MCAR假设，采用**迭代式贝叶斯岭回归**（Bayesian Ridge）或EM算法进行缺失值填补
      - （此方法与第一篇外文翻译文献做法一致 具体来说首先设置0.00001作为迭代容差 根据库函数（综合分析容差和数据集数据量大小）估计出最终收敛容差）

    - 对于高缺失率特征（>50%），考虑删除或创建缺失指示变量
      - （这个看情况 一般是放在第一步进行分析 具体需要根据其实际意义来分析如何处理）

  ### 1.2 目标变量分析
  - **类别分布**：统计流失用户与留存用户的比例，评估样本不平衡程度
  - **数据探索**：
    - 探索用户价值（CLTV）与流失倾向的关联
    - 分析在网时长分布特征，识别高风险时段
    - 评估不同用户群体的流失风险差异
      - 以上几点是看过往文献中提到的 之后如果有新的探索性分析另做分析

  ### 1.3 异常值检测与处理
  - **异常值识别**：使用Z-score、IQR等方法检测极端值
  - **异常值解释**：结合业务逻辑判断异常值的合理性（数据错误 vs 真实极端情况）
  - **处理策略**：
    - 数据错误：删除或修正
    - 真实极端值：保留或使用鲁棒缩放（Robust Scaling）降低影响
      - 同样是外文翻译第一篇 鲁棒缩放中位数和IQR对数据进行缩放 这两个指标对于异常值的敏感性不强 同时此缩放相当于让所有数据向着IQR区间的方向进行靠近 但是距离信息还是保留的 具有类似正则的间接性效果

  ### 1.4 数据标准化
  - **标准化必要性评估**：检查特征量纲差异和分布特征
  - **标准化方法选择**：
    - 鲁棒缩放（Robust Scaling）：对异常值不敏感，基于中位数和四分位距
    - 标准缩放（Standard Scaling）：基于均值和标准差，适用于近似正态分布
    - 最小-最大缩放（MinMax Scaling）：适用于需要固定范围的场景
      - (这个不一定用得上 我理解是做了统一量纲的作用 但是实际是不是能起到作用 可能需要消融实验分析)
  - **应用时机**：在特征工程完成后、模型训练前进行

  ## 二、特征工程

  ### 2.1 特征构造
  - **时序特征**：提取时间相关特征（如账龄、使用频率变化趋势等）
  - **交互特征**：构造有业务意义的特征组合（如价值-活跃度交叉特征）
  - **聚合特征**：基于用户行为历史计算统计量（均值、方差、趋势等）
  - **领域特征**：结合电信行业知识构造专家特征（如套餐匹配度、服务投诉率等）

  ### 2.2 特征选择
  - **Boruta算法**：
    - 创建影子特征（shadow features）：对原始特征进行随机排列生成同分布的随机特征
    - 迭代训练随机森林：比较原始特征与影子特征的重要性
    - 统计显著性检验：通过多次迭代判断特征是否显著优于随机特征
    - 特征确认/拒绝：保留显著特征，剔除不重要特征
  - **优势**：相比单次特征重要性评估，Boruta通过多次随机化降低偶然性，提供更稳健的特征选择结果

  ## 三、数据集划分与类别平衡

  ### 3.1 数据集划分
  - **划分策略**：按7:2:1比例（待定）划分训练集、验证集、测试集
  - **分层采样**：保证各数据集中流失用户比例一致

  ### 3.2 类别不平衡处理
  - **数据增强**：SMOTE及其变体（过采样少数类）（SMOTE只能在训练集上进行 如果应用在测试集上就无法证明模型是否可以识别真实分布了 失去了测试的意义了）
  - **欠采样集成**：结合Bagging构造多个平衡子数据集
  - **代价敏感学习（也是后文的部分内容）**：
    - 根据用户价值（CLTV）和流失风险（Churn Score）设置差异化的误分类代价
    - 在损失函数中对流失用户预测错误施加更高惩罚
    - 引导模型向高代价类别（流失用户）倾斜

  ## 四、模型构建

  ### 4.1 问题定义
  - **任务类型**：二分类问题（预测用户是否流失）
  - **评估指标**：综合考虑准确率、召回率、F1-score、AUC-ROC，重点关注流失类的召回率

  ### 4.2 基于SDAE的代价敏感模型

  #### 4.2.1 堆叠降噪自编码器（SDAE）
  - **架构设计**：多层编码器-解码器结构
  - **降噪机制**：**随机mask部分输入特征，强迫网络学习特征间的依赖关系**
  - **训练目标**：最小化重构误差（MSE），学习数据的鲁棒表示
  - **多层表示**：提取不同抽象层次的特征表示

  #### 4.2.2 集成欠采样策略
  - **Bagging集成**：
    - 从多数类（留存用户）中随机抽取多个子集
    - 每个子集与少数类（流失用户）组合，形成类别平衡的训练集
    - 构造多样化的训练数据，增强模型泛化能力
  - **特征表示融合**：将SDAE不同层的输出作为下游分类器的输入

  #### 4.2.3 代价敏感分类层
  - **全连接分类器**：在SDAE提取的特征表示基础上构建
  - **加权损失函数**：
    ```
    Loss = w₀ × BCE(y=0) + w₁ × BCE(y=1)
    其中 w₁ >> w₀，对流失用户的误判施加更高惩罚（或者这里的参数取值可以参考用户终身价值CLV的值）
    ```
  - **优化目标**：在保证整体性能的前提下，最大化流失用户的识别率

  ### 4.3 基准模型对比
  - **传统集成学习**：
    - Random Forest：集成决策树，处理非线性关系
    - XGBoost/LightGBM：梯度提升树，处理类别不平衡
    - AdaBoost：自适应提升，关注难分样本
  - **深度学习基线**：
    - 标准DNN：无降噪、无代价敏感的深度神经网络
    - SMOTE+DNN：数据层面处理不平衡的深度学习模型
  - **评估对比**：多维度比较各模型在测试集上的性能

  ## 五、模型解释与归因分析

  ### 5.1 消融实验设计
  系统性验证各组件的有效性：
  1. **基线模型**：无数据平衡、无代价敏感的标准DNN
  2. **数据平衡**：仅使用SMOTE或欠采样
  3. **Bagging集成**：仅使用欠采样Bagging，无代价敏感
  4. **代价敏感**：仅在损失函数中引入代价权重
  5. **数据平衡+Bagging**：组合数据层面和算法层面的不平衡处理
  6. **完整模型**：SDAE + Bagging + 代价敏感（提出的方法）

  分析各组件的边际贡献，验证设计合理性。

  ### 5.2 SHAP值分析
  - **全局解释**：
    - 特征重要性排序：识别对流失预测贡献最大的特征
    - Summary plot：展示特征值与SHAP值的关系
  - **局部解释**：
    - Force plot：解释单个样本的预测结果
    - Waterfall plot：展示各特征对个体预测的贡献方向和大小
  - **子群分析**：对比流失用户和留存用户的SHAP值分布差异

  ### 5.3 解释忠实性验证
  - **HotFlip方法**：
    - 基于梯度寻找特征的最小扰动方向
    - 识别能够翻转预测结果的关键特征变化
  - **SEA（Sufficient Explanation Approach）**：
    - 寻找充分解释集：最小的特征子集能够充分解释预测
    - 验证SHAP识别的重要特征是否确为决策关键
  - **一致性检验**：
    - 对比SHAP高重要性特征与HotFlip/SEA识别的关键特征
    - 若高度一致，则验证SHAP解释的忠实性
    - 若不一致，深入分析原因（特征交互、非线性效应等）

  ## 六、业务洞察与干预建议

  ### 6.1 流失风险因素识别
  - 基于SHAP分析总结主要流失驱动因素
  - 区分可干预因素（如服务质量）与不可干预因素（如年龄）
  - 识别高风险用户画像

  ### 6.2 精准干预策略
  - **分层干预**：
    - 高价值高风险用户：优先级最高，定制化挽留方案
    - 高价值低风险用户：常规维护
    - 低价值高风险用户：成本可控的批量干预
  - **干预时机**：基于模型预测的流失概率和SHAP分析的触发因素，确定最佳干预窗口
  - **干预措施**：针对关键流失因素设计针对性的产品、服务或营销策略

  ### 6.3 效果评估与迭代
  - **A/B测试**：对比干预组与对照组的实际流失率
  - **成本效益分析**：评估挽留成本与客户终身价值的关系
  - **模型更新**：根据干预反馈和新数据持续优化模型
